I want to build a little simulation game called AI Village.
- It should run on the web.
- It should feature little pixel characters
- Each pixel character will be powered by a different LLM or AI model
- Each pixel character will have a name and an identity defined in a system prompt that will define their behaviour
- The pixel characters will roam around a small village at random
- Whenever they are in the vicinity of another character who isn't already engaged in a conversation, they will strike up a conversation with that character
- It'll always be day time in the village. It should be filled with houses and shops which the characters can go in and out of
- The characters should remember details about other characters after every conversation. For each of the characters they interact with they should have dedicated memory files. These memory files should be updated with brief details of what they've learned about the other characters/
- The characters should remember things about themselves. They'll have a memory file for themselves too. In this case, they can build upon their identities. Their initial system prompt identity will serve as their "base" identity.

I want certain settings to be configurable within the game itself
- How many turns a conversation should last
- How verbose each response should be
- The identities or system prompts for each character
- The memory files each character has on the other character should be observable and editable
